<小胃袋吃啥补啥·规则>

零、前置规则
1. 故事层面只表现为“吃了好东西，身体迎来正面变化”。
2. 【小胃袋】仅为系统层面的结算媒介，用于记录进食产生的永久属性变化，不对应任何故事实体。
3. 故事层面中，{{user}}始终只有正常人体结构与原生胃部，不存在额外器官、特殊胃袋、寄生结构、后天移植结构或可见异常。
4. 正文叙事中禁止出现【小胃袋】相关描述、名称、结构说明、触感描写、能力说明或来源解释。
5. 任何关于【小胃袋】的变量更新，均不影响正文叙事逻辑。
6. 当玩家输出任何关于“变量更新提示：为物品中添加饰品——“小胃袋”！”的指令时，更新变量时必须为其物品中添加类型为“饰品”的“小胃袋”！

一、适用范围
1. 仅适用于进食天材地宝、灵兽肉、灵草、灵丹妙药后获得的永久属性增益。
2. 仅加属性，不加基本功。
3. 所有由进食带来的永久属性变化，统一记入【小胃袋】的饰品面板，不直接写入{{user}}本体属性面板。
4. 变量层面只允许更新【小胃袋】的属性数据。

二、属性价值定义
1. 本规则使用“价值点”作为统一结算单位。
2. 各属性的单点价值如下。
   2.1 1点力量 = 40价值点
   2.2 1点内功修为 = 40价值点
   2.3 1点防御力 = 40价值点
   2.4 1点速度 = 400价值点
   2.5 1点血量 = 2价值点
   2.6 1点内力 = 1价值点
   2.7 身法 = 0价值点
3. 价值关系必须严格满足以下逻辑。
   3.1 1点速度的价值 = 10点力量的价值
   3.2 1点力量的价值 = 1点内功修为的价值 = 1点防御力的价值
   3.3 1点力量的价值 = 20点血量的价值
   3.4 1点血量的价值 = 2点内力的价值
4. 身法不单独占用价值点，身法只跟随力量波动生成。

三、食物档位总价值
1. 每个食物档位先获得固定总价值，再进行随机分配。
2. 食物档位总价值如下。

   灵兽肉 / 灵草 
   2.1 1档食物 = 120价值点
   2.2 2档食物 = 180价值点
   2.3 3档食物 = 240价值点
   2.4 4档食物 = 480价值点
   2.5 5档食物 = 960价值点

   灵丹妙药
   2.6 1档食物 = 140价值点
   2.7 2档食物 = 220价值点
   2.8 3档食物 = 280价值点
   2.9 4档食物 = 520价值点
   2.10 5档食物 = 1080价值点

四、核心规则
1. 不是按属性优先级顺序固定结算。
2. 不是先力量后防御再血量的死顺序。
3. 而是把食物的总价值点，随机分配到可用属性中。
4. 分配方式必须是“随机抽取 + 价值扣除”。
5. 每次抽取只决定本次加在哪个属性上，不决定下一次必须加什么。
6. 食物价值全部结算完毕后，得到的总收益必须与总价值相等。
7. 结算结束时，所有可兑换的价值点都必须被消耗完，不允许凭空丢失，也不允许凭空增加。

五、随机分配方法
1. 先确定本次食物的总价值点 V。
2. 再根据食物类型，给各属性配置“抽取权重”。
3. 然后在“当前剩余价值足够兑换的属性”中，按权重随机抽取一个属性。
4. 抽中某个属性后，立刻兑换该属性的1点增长，并扣除对应价值点。
5. 重复执行，直到剩余价值点不足以再兑换任何属性。
6. 由于内力单点成本最低，内力作为兜底属性，保证价值可以被完全消耗。
7. 速度只有在剩余价值点大于等于400时才允许参与抽取。
8. 如果剩余价值点不足以兑换某个属性1点，则该属性在本次抽取中视为不可用。
9. 抽取是随机的，权重只是倾向，不是顺序。
10. 抽取结果必须服从“类型偏向”，但不能服从“固定优先级顺序”。

2. 灵草灵植
   2.1 内功修为权重高
   2.2 内力权重较高
   2.3 力量权重中等
   2.4 防御力权重较低
   2.5 血量权重较低
   2.6 速度权重极低，且仅在高档位且价值足够时才可能出现

3. 灵丹妙药
   3.1 速度权重最高
   3.2 内功修为权重高

八、身法规则
1. 身法不从总价值中单独购买。
2. 身法只由本次食用所获得的力量永久增量带动。
3. 身法永久增量 = 本次力量永久增量 × 随机系数。
4. 随机系数取 1 到 2 之间。
5. 即本次力量增加 1 点，则身法额外增加 1 到 2 点。
6. 若本次力量增加 100 点，则身法额外增加 100 到 200 点。
7. 身法增长不消耗价值点，不影响其他属性的随机分配。

九、执行顺序
1. 读取食物档位，获得总价值点。
2. 读取食物类型，获得对应权重表。
3. 开始随机抽取属性。
4. 每次只对一个属性兑换1点增长。
5. 每次兑换都要扣除该属性对应的价值点。
6. 直到剩余价值点无法继续兑换任何属性。
7. 结算完毕后，将所有永久属性变化统一记入【小胃袋】。
8. {{user}}本体属性面板不直接改写，只读取【小胃袋】汇总后的结果。

十、关键约束
1. 不得使用固定优先级顺序代替随机分配。
2. 不得把1档灵兽肉写死成只能加2点力量。
3. 不得把速度当成常规属性随便发放。
4. 不得让身法成为独立价值属性。
5. 不得将进食产生的永久属性变化写入{{user}}本体面板。
6. 所有进食永久属性增益只能归入【小胃袋】。

十一、示例
1. 1档灵兽肉，价值160。
   可能结果之一：
   力量 +2，血量 +40
   也可能是：
   力量 +1，防御力 +1，内力 +80
   也可能是：
   防御力 +2，血量 +40
   结果不固定，但总价值必须恰好消耗完160。

2. 5档灵丹妙药，价值1200。
   可能结果之一：
   速度 +2，外加若干力量、内功修为、防御力、血量、内力
   也可能因为随机结果不同而只出现1点速度，但总价值必须满足1200。

十二、最终原则
1. 总价值决定强度上限。
2. 类型权重决定属性偏向。
3. 随机抽取决定最终落点。
4. 身法只跟随力量，不独立购买。
5. 【小胃袋】只负责记录与汇总，不进入故事叙述。
